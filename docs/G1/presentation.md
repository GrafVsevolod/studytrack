# StudyTrack — трекер учёбы для студентов

## 1. Идея проекта

**StudyTrack** — это веб-приложение для студентов, которое помогает:

- планировать учебные цели;
- разбивать их на конкретные задачи;
- отмечать выполненное;
- видеть статистику по времени учёбы и прогрессу.

Цель — дать студенту простой инструмент, где видно, _сколько_ и _по каким предметам_ он реально занимается.

---

## 2. Проблема

- Учёба растянута по разным предметам, дедлайнам и заданиям.
- Студенты часто не ведут учёт времени, не видят общую картину.
- Сложно понять, хватает ли времени на важные предметы.
- Существующие трекеры либо слишком общие (под привычки), либо перегруженные.

---

## 3. Решение

**StudyTrack** предлагает:

- создавать учебные цели (например: «5 часов веб-разработки в неделю»);
- планировать задачи по датам и предметам;
- отмечать выполненные задачи;
- смотреть статистику: сколько учился, в какие дни, по каким целям.

Приложение работает в браузере и ориентировано на простой и понятный интерфейс.

---

## 4. Целевая аудитория

- Студенты вузов и колледжей.
- Особенно те, кто совмещает несколько предметов, курсов и проектов.
- Люди, которым удобно видеть прогресс в виде цифр и графиков.

---

## 5. Основные пользовательские сценарии

### 5.1. Первый вход

1. Пользователь открывает приложение по ссылке.
2. Видит экран приветствия / логина.
3. Авторизуется (упрощённая регистрация) или заходит в демо-режим.
4. Попадает на дашборд с целями и задачами.

### 5.2. Планирование учёбы

1. На дашборде пользователь создаёт **учебную цель**:
   - пример: «Веб-разработка — 5 часов в неделю».
2. Создаёт **задачи**, привязанные к цели:
   - пример: «Посмотреть лекцию по React», «Сделать домашку по TypeScript».
3. Задаче задаётся дата, примерная длительность и описание.

### 5.3. Отслеживание прогресса

1. После выполнения задачи пользователь отмечает её как «выполненную».
2. Прогресс по соответствующей цели обновляется.
3. На странице статистики он видит:
   - сколько часов занимался за неделю / месяц;
   - распределение по целям;
   - календарь с «продуктивными» днями.

---

## 6. Функции MVP

Минимальный набор фич для курсового проекта:

- Регистрация / вход (упрощённый вариант).
- Дашборд:
  - список целей;
  - список задач на сегодня / ближайшее время;
  - краткий блок статистики.
- Управление целями:
  - создание / редактирование / удаление.
- Управление задачами:
  - создание / редактирование / удаление;
  - отметка «выполнено».
- Страница статистики:
  - график по времени учёбы;
  - календарь-heatmap (через Canvas API).
- Адаптивный интерфейс (десктоп / мобильный).

## 7. Планируемый технологический стек

**Frontend:**

- React + TypeScript
- Redux Toolkit
- Vite
- Material UI (MUI)
- ESLint + Prettier (+ Stylelint)

**Тестирование:**

- Jest + React Testing Library
- Cypress (E2E)

**Backend:**

- Node.js + Express
- Лёгкая БД (SQLite/PostgreSQL)

**По требованиям курса:**

- Web Worker (расчёт статистики)
- Canvas API (календарь/heatmap)
- Google Analytics
- Sentry

---

## 8. Краткое резюме

StudyTrack — это учебный трекер, который поможет студентам структурировать обучение, видеть прогресс и эффективно планировать время.

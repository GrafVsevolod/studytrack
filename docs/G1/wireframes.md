Сами макеты нарисованы в Miro (ссылка будет ниже).

## 2. Список экранов

### 2.1. Экран приветствия / логина

Элементы:

- Логотип / название: **StudyTrack**
- Текст: короткое описание (1–2 строки).
- Кнопка **"Войти"**:
  - поля: email, пароль;
  - кнопка «Войти».
- Кнопка **"Попробовать демо"** (опционально).

Назначение: точка входа в приложение.

---

### 2.2. Дашборд

Элементы:

- Верхняя панель:
  - приветствие: «Привет, Имя»;
  - кнопки навигации: «Дашборд», «Статистика», «Выход».
- Блок **"Мои цели"**:
  - список целей;
  - для каждой: название, прогресс-бар, краткая статистика (например, «3/5 часов на этой неделе»).
  - кнопка «Добавить цель».
- Блок **"Задачи на сегодня"**:
  - список задач с чекбоксами;
  - кнопка «Добавить задачу».
- Маленький блок статистики:
  - «Учился X часов на этой неделе».

Назначение: основной рабочий экран.

---

### 2.3. Экран создания / редактирования цели

Элементы формы:

- Поле «Название цели» (input).
- Поле «Тип» (select: Учёба / Привычка).
- Поле «Цель по времени» (например, часов в неделю).
- Кнопки:
  - «Сохранить»
  - «Отмена»

Связь с flow:  
Пользователь попадает сюда с дашборда, нажав «Добавить цель» или «Редактировать».

---

### 2.4. Экран создания / редактирования задачи

Элементы формы:

- Поле «Название задачи».
- Поле «Цель» (выбор из списка целей).
- Поле «Дата» (datepicker).
- Поле «Планируемое время» (в минутах/часах).
- Поле «Описание» (textarea).
- Кнопки:
  - «Сохранить»
  - «Отмена»

Связь с flow:  
Пользователь попадает сюда с дашборда, нажав «Добавить задачу» или редактируя существующую.

---

### 2.5. Экран статистики

Элементы:

- Фильтры:
  - период (неделя / месяц);
  - выбор конкретной цели или «Все цели».
- Блок «График времени»:
  - линейный/столбчатый график (часы учёбы по дням).
- Блок «Календарь / heatmap»:
  - сетка дней месяца с уровнем интенсивности;
  - реализуется позже через Canvas API.
- Блок цифр:
  - «Всего часов за выбранный период»;
  - «Количество дней с учёбой подряд (streak)».

---

## 3. Пользовательский flow по макетам

1. Пользователь открывает приложение → видит **экран приветствия / логина**.
2. Нажимает **«Войти»** → попадает на **дашборд**.
3. На дашборде:
   - видит свои цели и задачи на сегодня;
   - может нажать **«Добавить цель»** → попадает на экран создания цели;
   - может нажать **«Добавить задачу»** → попадает на экран создания задачи.
4. После сохранения цели/задачи пользователь возвращается на **дашборд**.
5. Отмечая задачи выполненными, он обновляет прогресс целей.
6. Нажимая в шапке **«Статистика»**, пользователь попадает на экран статистики, где видит графики и календарь.

Этот flow отражён стрелками и подписями на доске Miro.
